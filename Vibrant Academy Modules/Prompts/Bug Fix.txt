══════════════════════════════════════════════════════════════════════════════
DEBUGGING METHODOLOGY — EXECUTE IN ORDER
══════════════════════════════════════════════════════════════════════════════

PHASE 1: REPRODUCTION & ISOLATION
─────────────────────────────────
For every bug/glitch found:
1. Identify exact trigger conditions (inputs, state, timing, environment)
2. Determine if it's deterministic or intermittent (heisenbug)
3. Isolate minimal reproduction case
4. Verify the bug exists in current code (don't trust reports blindly)

PHASE 2: ROOT CAUSE ANALYSIS
────────────────────────────
Trace through execution flow:
- Follow data transformation from input to output
- Check every conditional branch — is the logic correct?
- Verify all state mutations — who writes, who reads, when?
- Inspect async flow — promise chains, callbacks, event emitters
- Examine timing — race conditions, stale closures, execution order
- Check environment assumptions — Node version, browser APIs, OS differences

PHASE 3: FIX & VERIFY
─────────────────────
- Apply minimal surgical fix that addresses root cause, not symptoms
- Ensure fix doesn't break existing functionality (regression test)
- Verify edge cases around the fix
- Confirm no new bugs introduced

══════════════════════════════════════════════════════════════════════════════
GLITCH CATEGORIES — CHECK EVERY FILE FOR:
══════════════════════════════════════════════════════════════════════════════

【LOGIC & CONTROL FLOW BUGS】
□ Wrong boolean logic (&& vs ||, ! misplaced, === vs == issues)
□ Off-by-one errors in loops, array access, pagination
□ Missing break/continue/return causing fall-through bugs
□ Switch statement missing cases or default
□ Ternary operators with wrong precedence
□ Short-circuit evaluation causing skipped side effects
□ Null/undefined checks in wrong order (can't access prop of null)
□ Truthy/falsy checks catching unexpected values (0, "", false)

【STATE & DATA BUGS】
□ Variable shadowing causing wrong value to be used
□ Stale closures in callbacks/async functions
□ Mutating shared state unexpectedly (objects/arrays passed by reference)
□ Race conditions in state updates (read-modify-write not atomic)
□ Deep vs shallow copy confusion
□ Date/timezone arithmetic errors
□ Floating point precision issues
□ Unicode/string encoding mishandling
□ JSON serialization edge cases (circular refs, undefined, NaN, Infinity)

【ASYNC & CONCURRENCY BUGS】
□ Unawaited promises (fire-and-forget causing unhandled rejections)
□ Promise chain missing .catch() or try/catch
□ Async/await in non-async context
□ Race conditions in parallel execution (Promise.all without coordination)
□ Callback hell with error handling gaps
□ Event listener leaks (added but never removed)
□ Stream handling errors (not waiting for 'end', 'error', 'close')
□ Timer leaks (setInterval without clearInterval)
□ Microtask vs macrotask ordering assumptions

【API & I/O BUGS】
□ HTTP client not handling timeout/network errors
□ Retrying failed requests without backoff (thundering herd)
□ Assuming response format without validation
□ Not checking HTTP status codes before parsing body
□ File system operations without error handling (ENOENT, EACCES, EMFILE)
□ Database connection not released on error
□ External service downtime not handled gracefully

【REACT/FRONTEND SPECIFIC GLITCHES】
□ useEffect dependency array missing values → stale state
□ useEffect cleanup function missing → memory leaks
□ State updates based on previous state done wrong (not using function form)
□ Event handlers not preventing default when needed
□ Key prop issues causing component remount bugs
□ Form inputs controlled/uncontrolled mismatch
□ Refs accessed before mount
□ setState called after unmount (can't perform React state update... warning)

【MEMORY & RESOURCE BUGS】
□ Memory leaks: closures holding references to large objects
□ Unclosed database connections, file descriptors, sockets
□ Event emitter maxListeners exceeded
□ Large arrays/objects not being released (garbage collection blocked)
□ Recursive functions without base case or depth limit
□ Buffer/Buffer.alloc vs Buffer.from confusion

【ENVIRONMENT & CONFIG GLITCHES】
□ Environment-specific code (dev vs prod) causing different behavior
□ Hardcoded paths that break on different OS
□ Case sensitivity issues (macOS vs Linux file systems)
□ Timezone/DST bugs (new Date() vs UTC methods)
□ Feature detection vs browser sniffing done wrong

══════════════════════════════════════════════════════════════════════════════
DEBUGGING TACTICS — USE AS NEEDED:
══════════════════════════════════════════════════════════════════════════════

TACTIC: Binary Search Debugging
- Comment out half the code → bug still there? → other half contains bug
- Repeat until isolated

TACTIC: Rubber Duck / Trace Logging
- Add console.log at every significant step showing:
  - Input values and types
  - Branch taken (which if/else)
  - State before and after mutation
  - Return values
- Remove all debug logs before final output

TACTIC: Assume Nothing
- Don't trust that a function returns what it says
- Verify every external call's actual behavior
- Check that imports are what you expect (not a different version)

TACTIC: Boundary Testing
- Test with: empty arrays, null, undefined, 0, -1, MAX_SAFE_INTEGER, 
  strings with special chars, unicode, very long strings, circular objects

TACTIC: Concurrency Simulation
- Add artificial delays (setTimeout, sleep) to expose race conditions
- Force promise resolution order to change
